# The Motorcity Mouth

A Detroit satirical news website, styled like a supermarket-checkout tabloid. Reads like real gossip. Is not.

**Live site:** themotorcitymouth.com  
**Hosting:** Firebase Hosting (project: `motorcity-mouth`)  
**Contact:** motorcitymouth@spaceflotsam.com

---

## How It Works

The site has two layers:

1. **`public/index.html`** — the main SPA. Hash-based routing, single file, no framework. Handles the front page, section pages, and article previews.
2. **`public/articles/*.html`** — static article pages generated by `scripts/generate-articles.js`. These are real, standalone URLs — they have their own `<head>` SEO, share buttons, and structured data. When someone shares a link or a search engine crawls an article, this is what they see.

Navigation on the SPA is hash-based — clicking around changes the URL hash (e.g. `#politics`, `#article/kwame-kilpatrick-2029`) and JavaScript re-renders the page content without any full reload. Article links always go to the real static pages (`/articles/slug.html`), not hash routes.

### Pages / Routes (SPA)

| Hash | What renders |
|------|-------------|
| `#front` (default) | Front page with lead story, sidebar, lower grid, horrorscope preview, and live Detroit weather |
| `#politics` | Section page listing all Politics articles |
| `#world` | World News section |
| `#local` | Local section |
| `#trending` | Trending section |
| `#columns` | Weekly Columns landing — currently shows the Dave's Psychic Hotline card |
| `#horrorscopes` | Full page: Dave's Psychic Hotline (horoscopes) for all 12 signs |
| `#article/<id>` | Full article page (SPA version — also available at `/articles/<id>.html`) |

### Live Features

- **Doom Gauge (weather widget)** — front page pulls current Detroit conditions from the free [Open-Meteo API](https://open-meteo.com/) (no API key required). Coordinates are hardcoded to Detroit (42.3314°N, 83.0458°W). Falls back gracefully if the fetch fails.
- **Dave's Psychic Hotline preview** — front page automatically detects today's zodiac sign and shows the matching blurb from the `HORRORSCOPES` data object.
- **Live date** — the top strip shows today's date, set by JavaScript on load.
- **Cover lines** — the masthead teaser strip pulls 2–3 real front-page headlines at load time (tabloid corner-blurb convention), rather than being hand-written.
- **Share buttons** — every article page has share buttons for Copy Link, Facebook, X/Twitter, Instagram (via Web Share API on mobile, falls back to copy), and Text Message (SMS). Buttons appear in a compact icon-only row above the hero image and a labeled row below the article body.

## Adding or Updating an Article

All articles are in the `ARTICLES` array in `public/data/articles.js`. Each article is a JavaScript object:

```js
{
  id:           'unique-slug',        // URL slug — letters, numbers, hyphens only
  headline:     'Article Headline',
  deck:         'Subheadline here',   // optional — leave '' to omit
  section:      'politics',           // see valid sections below
  featured:     true,                 // true = lead story on front page (only one at a time)
  frontPage:    true,                 // true = eligible for front page sidebar and lower grid
  byline:       'By Staff Reporter',
  date:         'April 18, 2026',
  lede:         'Opening paragraph text…',  // used in teasers/cards
  body:         `<p>Full article HTML…</p>`, // full text, use <p> tags
  image:        'images/my-image.jpg',       // path relative to public/ — or null
  imageCaption: 'Caption text here.',        // only shown when image is set
}
```

**Valid section values:**

| Section | Where it appears |
|---------|-----------------|
| `politics` | Politics section |
| `world` | World News section |
| `local` | Local section |
| `trending` | Trending section |
| `horrorscopes` | Dave's Psychic Hotline (special — see below) |

**Front page layout rules:**
- Exactly one article should have `featured: true` — this becomes the large lead story.
- The sidebar shows up to 4 articles that have `frontPage: true` and are not the featured story, in array order.
- The lower "More From Today's Edition" grid shows up to 3 articles with `frontPage: true` that didn't make the sidebar.
- Order in the `ARTICLES` array controls what surfaces where — articles listed earlier rank higher.

**To remove an article:** delete its object from the `ARTICLES` array.

**Images:** drop the file into `public/images/` and set `image: 'images/filename.jpg'` in the article object. See image guidelines below.

### After editing articles.js — regenerate static pages

Any time you add, edit, or remove an article in `public/data/articles.js`, run the generator to rebuild the static article pages, sitemap, and robots.txt:

```bash
node scripts/generate-articles.js
```

This regenerates all files in `public/articles/`, `public/sitemap.xml`, and `public/robots.txt`. Commit the changes and deploy.

If you also want the archived broadsheet pages to stay in sync with content edits (not required for the live site), run `node scripts/generate-articles-alt.js` too — it only writes into `public/articles-alt/` and never touches the sitemap or robots.txt.

## Image Guidelines

Images are displayed in several contexts with fixed dimensions. Supply images that are at least as large as the display sizes below — smaller images will be upscaled and look blurry.

| Context | Display size | Aspect ratio | Notes |
|---------|-------------|-------------|-------|
| Lead story (front page) | ~800px wide | **16:9** | `max-height: 440px` cap applied; wrapped in the "evidence" photo frame (halftone overlay + corner stamp) |
| Sidebar stories | ~260px wide | any (cropped to **130px tall**) | `object-fit: cover` — center of image is shown |
| Lower grid ("Also Spotted This Week") | ~300px wide | any (cropped to **4:3**) | Rendered as a tilted Polaroid card, `object-fit: cover` |
| Article page hero (in-app SPA) | full column width | any (cropped to **480px tall**) | `object-fit: cover` |
| Article page hero (static `/articles/*.html`) | full column width | any (cropped to **440px tall**, 260px on mobile) | `object-fit: cover` |
| Section card thumbnails | ~360px wide | any (cropped to **16:9**) | `object-fit: cover` |

**Recommended minimum source resolution:** 900px wide for article images; 1200px wide for the lead story image.

**Supported formats:** `.jpg`, `.jpeg`, `.png`, `.avif`, `.svg`

## Updating Dave's Psychic Hotline (Horoscopes)

The horoscope content lives in a `HORRORSCOPES` object in `public/data/articles.js`, exported alongside `ARTICLES`. It has two parts:

- **`intro`** — HTML string shown at the top of the Dave's Psychic Hotline page. Dave's intro paragraphs.
- **`signs`** — Array of 12 sign objects. Each has `symbol`, `name`, `dates`, `key`, and `text`. Update `text` to change that sign's weekly blurb. The front page automatically shows the current sign's blurb based on today's date.

Both `index.html` and `indexalt.html` read this same object — an edit updates the live site and the archived design at once.

### Horoscope Image

`public/images/horrorscopes.jpg` is the zodiac wheel illustration used in three places:

1. **Weekly Columns section page** — the Dave's Psychic Hotline card image
2. **Front page bottom strip** — small banner above the sign preview
3. **Dave's Psychic Hotline page** — full-width banner between the masthead and Dave's byline

To swap the illustration, replace `public/images/horrorscopes.jpg` with a new image (keep the same filename, or update the references in the JS render functions: `renderSection`, `renderFront`, and `renderHorrorscopes`, in both `index.html` and `indexalt.html`).

## Deploying

The site is hosted on Firebase. To deploy after making changes:

```bash
# If you edited articles (public/data/articles.js), regenerate static pages first
node scripts/generate-articles.js

# Install Firebase CLI if you haven't
npm install -g firebase-tools

# Log in
firebase login

# Deploy from the project root
firebase deploy
```

Firebase serves everything in the `public/` folder. That's it — one command, site is live.

## Share Buttons

Every static article page (`/articles/*.html`) includes two share bars:

- **Top bar** — compact icon-only buttons above the hero image
- **Bottom bar** — labeled buttons (icon + text) below the article body

Available share targets on each article:

| Button | Behavior |
|--------|----------|
| **Copy Link** | Copies the article's canonical URL to the clipboard. Button briefly shows "Copied!" |
| **Facebook** | Opens `facebook.com/sharer` with the article URL pre-filled |
| **Twitter / X** | Opens `twitter.com/intent/tweet` with URL + headline |
| **Instagram** | On mobile (or any browser supporting the Web Share API), opens the native OS share sheet — the user can pick Instagram, WhatsApp, etc. On desktop without Web Share API support, falls back to copying the link. |
| **Text** | Opens the default SMS app with a pre-filled message containing the headline and URL |

Share buttons are generated by `scripts/generate-articles.js` — the URL and headline are hardcoded into each static page so they always share the correct canonical link.

## SEO

Each static article page is fully optimized for search and social:

- **`<title>`** — `Headline — The Motorcity Mouth`
- **`<meta name="description">`** — pulled from the article deck, lede, or first 160 characters of body
- **`<meta name="keywords">`** — Detroit, satire, The Motorcity Mouth, Detroit news, [section]
- **`<link rel="canonical">`** — the article's permanent URL
- **Open Graph tags** — `og:type`, `og:title`, `og:description`, `og:image`, `og:url`, `og:locale`, `og:image:alt`, `article:published_time`, `article:modified_time`, `article:section`, `article:author`, `article:tag`
- **Twitter Card tags** — `summary_large_image` card with title, description, and image
- **JSON-LD structured data** — `NewsArticle` schema with `mainEntityOfPage`, `datePublished`, `dateModified`, `inLanguage`, `isAccessibleForFree`, `image` as `ImageObject`, `author`, `publisher` (with logo as `ImageObject` and `sameAs` Facebook link), `articleSection`, `genre`, and `keywords`
- **`<meta name="theme-color">`** — matches the paper background for mobile browser chrome
- **`sitemap.xml`** — auto-generated with all article URLs and pub dates
- **`robots.txt`** — allows all crawlers, points to sitemap

All SEO metadata is generated from the article data in `public/data/articles.js` — editing an article and re-running `generate-articles.js` keeps everything in sync.

## Design Notes

The live site uses Google Fonts (loaded via CDN — no download needed):
- **Anton** — masthead and all headlines (bold condensed tabloid display face)
- **Oswald** — labels, nav, metadata, tags
- **Libre Baskerville** — body text
- **Permanent Marker** — handwritten-style photo captions ("evidence" frame)
- **Barlow Condensed** / **IM Fell English** — retained for inline article-body HTML that references them (a couple of older articles have hand-styled pull quotes using these)

Color palette is defined as CSS variables at the top of the `<style>` block:
- `--paper`: `#f7f3e8` (warm off-white background)
- `--ink`: `#111111` (near-black text, masthead background)
- `--red`: `#d81324` (tabloid red — tags, flags, stamps, accents)
- `--yellow`: `#ffcf00` (masthead wordmark, label chips)
- `--muted`: `#4a4a4a` (secondary text)
- `--light-rule`: `#c9c0aa` (column dividers and card borders)

Real article photos are wrapped in an "evidence" frame (`.evi-photo`): a subtle halftone-dot overlay for a cheap-newsprint feel, an optional corner "Exclusive" stamp, and a marker-font caption — applied consistently across the lead image, sidebar thumbnails, section grids, lower-grid Polaroids, and article hero images.

The layout is fully responsive with breakpoints at 800px (tablet) and 540px (mobile). Nav is always visible (no hamburger) and wraps into rows on narrow screens, with dropdown support for the Weekly Columns sub-menu.

## Archived Design: Broadsheet (`indexalt.html`)

`public/indexalt.html` is the site's previous design — a classic broadsheet-newspaper layout (Libre Bodoni masthead, Playfair Display headlines, deep red `--accent: #8b1a1a`). It was retired in favor of the tabloid direction now live at `index.html`, and is kept as a reference/rollback copy rather than deleted.

It is a fully independent, parallel build sharing only the data layer:

- **Same data, same router shape.** `indexalt.html` loads `public/data/articles.js` and implements its own copy of the hash-based router and render functions (front page, all 5 sections, in-app article view, horoscope page, contact, customer service). Editing an article in `data/articles.js` updates both designs automatically — there's no separate content to maintain.
- **`public/articles-alt/*.html`** is the broadsheet counterpart to `public/articles/*.html` — one static page per article, generated by `scripts/generate-articles-alt.js`:

  ```bash
  node scripts/generate-articles-alt.js
  ```

  This only ever writes into `public/articles-alt/` — it never touches `public/articles/`, `sitemap.xml`, or `robots.txt`, which belong to the live site.
- **Marked as archived, not live:**
  - A rotating "Archived Design" corner badge marks every page (front end and static article pages).
  - `<meta name="robots" content="noindex, nofollow">` keeps it out of search results, so it can't compete with the live site for rankings.
  - No Google Analytics tag and no AdSense script — avoids double-counting traffic or serving stray ads from a non-production page.
  - Internal links point at `indexalt.html` / `articles-alt/` throughout, so browsing the archived design never accidentally drops you onto the live tabloid pages (or vice versa).

**If you ever want to roll back** to the broadsheet: swap the file/directory names back (`index.html` ↔ `indexalt.html`, `articles/` ↔ `articles-alt/`, `generate-articles.js` ↔ `generate-articles-alt.js`), re-add the full production `<head>` (GA tag, SEO/OG/structured-data meta, AdSense, `index, follow` robots) and the sitemap/robots.txt generation block to whichever script is being promoted, and strip that scaffolding — plus add the noindex badge — from whichever is being demoted. That's the same procedure used to promote the tabloid in the first place.
